    How to use KekaScraper

    What it does

    KekaScraper searches Google Maps for hostels in the cities you specify, extracts their public details (name, address, phone, website, rating), and optionally enriches results by scanning each hostel's website for contact information like email addresses, booking pages, and social media profiles.

    Getting started

    1. Enter one or more cities (one per line, or separated by semicolons).
    2. Set the maximum results per city (default: 20).
    3. Choose your output format: JSON, CSV, or both.
    4. Click Start scrape and watch the activity log for progress.

    Options explained

    Browser
    Which Chromium-based browser to use for scraping. "Auto" tries Edge first, then Chrome, then bundled Chromium (dev builds only).
    Show the browser while scraping
    Opens a visible browser window so you can watch the scraping process in real time. Useful for debugging or verifying that the scrape is progressing correctly.
    Enrich results from each hostel website
    After collecting listings from Google Maps, visits each hostel's website to find email addresses, contact forms, decision-maker details, social media links, and room/bed counts. This adds significant detail but makes the run slower.
    Parallel cities
    Process multiple cities simultaneously (1 to 5). Higher values finish faster but use more memory and CPU. Start with 1 if unsure.
    Parallel listings
    Extract multiple listings at once within a single city (1 to 6). Keep this low to reduce the chance of being rate-limited by Google Maps. Values above 3 may cause blocks without a proxy.
    Proxy
    Route all browser traffic through an HTTP or SOCKS5 proxy server. Format: http://user:pass@host:port or socks5://host:port.

    Output formats

    JSON — Machine-readable format, ideal for further processing, importing into databases, or feeding into other tools.

    CSV — Spreadsheet-compatible format that opens directly in Excel, Google Sheets, or Numbers. One row per hostel.

    Files are saved to the output folder shown in the header. You can open it directly from the status panel.

    Tips

    • Start with a small run (1 city, 5–10 results) to verify everything works before scaling up.
    • If a scrape is interrupted, re-running the same cities will resume from the last checkpoint automatically.
    • All results stay on your machine. No data is sent to any external service or cloud backend.
    • The dashboard shows a live preview. Use the exported file or Load JSON for the full dataset.
